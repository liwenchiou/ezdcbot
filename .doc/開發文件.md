# 軟體需求規格書：ezdcbot (v1.0)

## 1. 專案背景與目標

- **名稱**：ezdcbot (Easy Discord Bot)
- **目標**：提供一個極致輕量、零依賴 (Zero Dependency) 的 npm 套件，讓開發者能以最少的程式碼，實現 Discord 頻道訊息推送功能。
- **解決痛點**：克服 discord.js 安裝龐大、架構複雜且不適合 Serverless (如 Vercel, AWS Lambda) 等無狀態環境的限制。

## 2. 核心技術架構

- **通訊協定**：Discord REST API (HTTPS POST)
- **執行環境**：Node.js
- **依賴需求**：無 (Zero Dependencies)，利用原生 fetch API，極大化輕量化優勢。

## 3. 功能需求 (Functional Requirements)

### 3.1 訊息推送 (The `push` Method)

- **基本需求**：支援發送純文字訊息。
- **進階需求**：支援發送 Embed 物件（格式化圖文）。
- **錯誤處理**：若 API 回傳錯誤（如權限不足、頻道 ID 無效、限流），需回傳清晰的錯誤訊息（包含 HTTP Status Code）。

### 3.2 環境設定 (Configuration)

- **靜態設定**：支援在初始化時傳入 Token。
- **動態設定**：支援在方法呼叫時傳入 Token (適合多 Bot 環境)。

## 4. API 介面定義 (API Reference)

### 4.1 初始化

```javascript
const ezdcbot = require('ezdcbot')('YOUR_BOT_TOKEN');
```

### 4.2 發送訊息

```javascript
/**
 * @param {string} channelId - 目標頻道 ID
 * @param {string|object} content - 訊息內容 (文字或 Embed 物件)
 */
await ezdcbot.push('CHANNEL_ID', 'Hello, World!');
```

## 5. 非功能需求 (Non-Functional Requirements)

- **輕量化**：套件安裝後的 `node_modules` 體積必須在 50KB 以下。
- **效能**：平均 API 發送時間需在 300ms 以內。
- **安全性**：絕不儲存、記錄或輸出使用者的 Bot Token。

## 6. 使用者體驗流程 (DX Flow)

- **安裝**：使用者執行 `npm install ezdcbot`。
- **整合**：開發者在程式碼中 `require` 套件，並輸入 Token。
- **執行**：呼叫 `.push()` 方法，系統自動處理 Authorization Header 與 JSON 格式化。

## 7. 未來擴充規劃 (Roadmap)

- **Phase 1**：純文字訊息推送 (MVP 版本)。
- **Phase 2**：支援 Embed 訊息與檔案上傳 (Multipart form-data)。
- **Phase 3**：速率限制處理 (Rate Limit Queue)，自動偵測 429 狀態碼並延遲重試。
- **Phase 4**：TypeScript 支援，提供完整型別定義檔 (`.d.ts`)。

## 8. 開發建議事項

- **錯誤偵測**：Discord API 對於無效 Token 或無效 Channel ID 的回應不同，建議在 `push` 方法中增加檢查邏輯，避免使用者輸入錯誤時毫無頭緒。
- **文件撰寫**：因為你的目標是「簡便」，`README` 的教學範例必須直接放在首頁，並明確列出「為什麼這比 discord.js 更好用」的優點。
- **測試**：建立一個簡單的 `test.js`，讓使用者安裝後能直接跑一次確認連線是否正常。
